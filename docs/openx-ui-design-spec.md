# OpenX Agent 运行台 UI 设计规范

> 当前实现对齐 `apps/web`。视觉为 Apple-inspired 清爽控制台；信息架构为 **Agent 运行台**（非四宫格）。

## 1. 设计目标

OpenX 是「工头层」控制台：用户制定目标、验收成果、要求返工；OpenX Coach 整理意图为可执行任务，**不**替代 CLI 写代码。

界面气质：**清爽产品控制台** — 浅色背景、圆角卡片、玻璃感顶栏；保留工头播报与实时日志，强调「运行中」心智。

## 2. 信息架构（Agent 运行台）

```text
┌────────────┬────────────────────────────────────────────────────────┐
│ 侧栏业务功能 │ 顶栏：执行范围（全部/模拟/本机）· 待命 · 智能模式      │
│            ├────────────────────────────────────────────────────────┤
│ 首页        │ 播报细条：OpenX 正在推进/提醒确认                     │
│ 正在推进    ├──────────────────────┬─────────────────────────────────┤
│ 等我确认    │ 目标业务区             │ 我的助手                         │
│ 我的助手    │ 五态筛选·进度·验收     │ 对话·整理目标·查看进展·输入框    │
│ 设置        │                      │                                 │
│ ＋新目标    │                      │                                 │
├────────────┴──────────────────────┴─────────────────────────────────┤
│ 最新动态：当前目标/全部 · 类型筛选 · 暂停滚动 · 可展开                 │
└──────────────────────────────────────────────────────────────────────┘

Modal: 新目标 — 草稿 · Coach refine · 执行器 · 保存草稿/开始执行
```

## 3. 区域要点


| 区域   | 必含元素                                                                  |
| ---- | --------------------------------------------------------------------- |
| 侧栏   | 分组：总览（首页）/ 业务功能（正在推进·等我确认，带数量角标）/ 助手与配置；工作目录摘要；待命状态；底部「＋ 新目标」         |
| 顶栏   | 执行范围筛选；待命状态；智能/基础模式徽章                                                 |
| 播报   | `narration.append` 转成用户可读播报；悬停暂停；保留最近 5 条；`aria-live="polite"` 朗读最新一条 |
| 首页   | 全部目标 + 助手同屏，作为默认总览                                                    |
| 正在推进 | 业务入口，只展示 `running` 目标，强调下一步与进度                                        |
| 等我确认 | 业务入口，只展示 `awaiting_review` 目标，强调确认完成/还要修改                             |
| 我的助手 | 独立助手页，用于整理目标、询问进展、应用整理结果；订阅 `coach.reply` SSE 实时追加回复                  |
| 最新动态 | SSE 状态；默认聚焦当前目标；展开后：当前/全部、类型筛选、暂停滚动                                   |
| 设置   | 工作目录选择/粘贴；默认执行器、Pi 路径；autoExecute/notify；Zen 免费预设；执行器检测               |

## 5. 关键交互流

1. **选择工作目录**：设置页选择/粘贴项目文件夹 → 保存后后续目标默认在该目录推进
2. **新目标**：侧栏或任务区 → Modal 草稿 → Coach refine → 选 Mock/Pi → 创建/启动 → 播报 + 日志联动
3. **达标/返工**：`awaiting_review` 且选中时显示；返工可填原因；完成后 narration 播报
4. **CLI 筛选**：顶栏切换执行器，任务列表与统计同步过滤（开发/调试向，非最终用户心智）
5. **Zen 免费**：设置一键预设 → 顶栏 `Zen · model`；超额中文提示 + 规则回退

## 7. Smart Cabin · 柔性主屏（HyperOS 对齐）

> 参考 [小米澎湃智能座舱](https://www.xiaomiev.com/smartcabin)：主屏为 **可组合、可 resize 的卡片桌面**，而非对话 / 看板 / 预览三列常驻。

### 7.1 布局分层

```text
┌─────────────────────────────────────────────────────────────┐
│ Smart Strip（全局状态：场景、WO、待验收、执行器）              │
├──────────┬──────────────────────────────────────────────────┤
│ 左轨      │ ★ Flexible Canvas（柔性主屏）★                    │
│ 任务索引  │ 1～2 张 Widget 卡片 · 拖边界 resize · Pin/Unpin   │
│ ~18% 固定 │                                                  │
├──────────┴──────────────────────────────────────────────────┤
│ Foreman Dock（切换场景预设，不是再加一列）                     │
└─────────────────────────────────────────────────────────────┘
```

- **左轨** `TaskIndexCard`：WO 列表 + 筛选，固定宽度，不参与柔性组合。
- **柔性主屏** `FlexibleCanvas`：Widget 卡片画布，组合与比例由底栏模式决定。
- **Foreman Dock**：切换桌面预设；快捷验收 / 推进 / 返工。

### 7.2 Widget 注册表


| Widget ID   | 组件                        | 说明         |
| ----------- | ------------------------- | ---------- |
| `chat`      | `ChatPanel`               | 工头对话       |
| `tasks`     | `GoalsWorkspace`          | 任务看板 / 列表  |
| `artifacts` | `PreviewRail`             | 交付物 / Diff |
| `review`    | `AwaitingReviewCompanion` | 待验收快捷      |
| `fleet`     | `ConsoleFleetPanel`       | 执行器（仅调度台）  |


### 7.3 底栏模式 → 默认画布组合


| Dock 模式     | 场景   | 主 Widget | 辅 Widget（可 Pin） | 默认比例      |
| ----------- | ---- | -------- | --------------- | --------- |
| `chat`      | 派单桌面 | 对话       | 待验收             | 62% / 38% |
| `tasks`     | 调度桌面 | 任务看板     | —               | 100%      |
| `artifacts` | 施工桌面 | 对话       | 产物预览            | 52% / 48% |
| `fleet`     | 调度   | 执行器      | —               | 100%      |


规则：

1. 画布上 **最多 2 张 Widget 并排**；第三张必须 Unpin 或通过 Dock 切换。
2. 辅 Widget 标题栏 **固定 / 取消固定**；Unpin 后主 Widget 占满，底部可一键恢复。
3. 分隔线拖拽比例写入 `localStorage`（`openx.flex.{scope}.{dockMode}.ratio`）。
4. Pin 状态写入 `openx.flex.{scope}.{dockMode}.pin`。

实现：`apps/web/src/lib/flexible-desktop.ts`、`components/smart-cabin/FlexibleCanvas.tsx`。
