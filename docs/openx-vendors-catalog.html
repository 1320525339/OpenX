你点名的 5 个 · 逐项结论

1. Pi Agent Executor / SDK 本地 vendors/pi

是什么：Mario Zechner 发起的开源 coding agent 工具链；主仓已迁移至 earendil-works/pi（原 badlogic/pi-mono）。Monorepo：pi-coding-agent、pi-agent-core、pi-ai、pi-tui。

和 OpenX 的关系：非常适合做本机 Executor。四种模式：交互 TUI、print/JSON、RPC（stdin/stdout JSONL）、SDK 嵌入 — 工头层只需调 RPC/SDK，不必重做工具循环。

建议阅读的脚手架

  • vendors/pi/packages/coding-agent/README.md — RPC / SDK 入口
  • vendors/pi/packages/coding-agent/docs/rpc.md
  • vendors/pi/packages/coding-agent/examples/sdk/
  • vendors/pi/packages/agent/ — agent runtime
  • vendors/pi/packages/ai/ — 统一多 Provider LLM API

注意：热门 fork oh-my-pi（子代理、Rust 加速）未克隆，需要时可单独 shallow clone。

安装体验：npm i -g @earendil-works/pi-coding-agent

2. OpenCode Executor + Server vendors/opencode

是什么：SST/anomalyco 旗下超热门开源 coding agent（~168k stars）。终端 TUI + 中央 server + 桌面端 + VS Code 扩展；75+ 模型 Provider；内置 build/plan 主代理与 @general 子代理。

和 OpenX 的关系：可作为 Executor（调其 server/session API），或学其 packages/opencode 的会话与工具循环。不适合整项目 fork — 体量大、产品完整度高，易变成「又一个 OpenCode」。

建议阅读路径

  • vendors/opencode/packages/opencode/ — 主 CLI + server
  • vendors/opencode/packages/server/
  • vendors/opencode/packages/sdk/js/
  • vendors/opencode/packages/web/src/content/docs/agents.mdx

安装：npm i -g opencode-ai@latest

3. OpenClaude Executor CLI vendors/openclaude

是什么：社区项目，把 coding-agent 工作流扩展到 OpenAI / Gemini / Ollama / Codex OAuth 等；终端优先；带 VS Code 扩展与 /provider 配置。

和 OpenX 的关系：典型「一个 CLI 多模型」执行面；工头层对用户隐藏品牌后，可把它当作 Executor 之一。许可证为 NOASSERTION / 见 LICENSE 文件，商用前需法务过一遍。

建议阅读路径

  • vendors/openclaude/src/ — 核心 runtime
  • vendors/openclaude/bin/openclaude
  • vendors/openclaude/package.json — SDK export ./sdk

安装：npm i -g @gitlawb/openclaude

声明：与 Anthropic 无官方关系；名称含 Claude 需注意商标说明（仓库 README 已写）。

4. DeepSeek-TUI → CodeWhale Executor · Rust vendors/codewhale

重要：项目已改名为 CodeWhale（codewhale / codewhale-tui）。deepseek-tui npm 已弃用，二进制仅为过渡期 shim。

是什么：Rust 终端 coding agent，面向 DeepSeek V4；流式 reasoning、审批门、子代理、MCP、任务队列、LSP 诊断反馈。

和 OpenX 的关系：执行层；release 笔记提到 Goal Tools，与你们 Goal 模型有概念重合，可读其任务/验证设计。集成方式：子进程调用 codewhale 非交互模式（需读 docs 是否支持 headless/JSON）。

建议阅读路径

  • vendors/codewhale/README.md · README.zh-CN.md
  • vendors/codewhale/docs/REBRAND.md
  • vendors/codewhale/docs/INSTALL.md

安装：npm i -g codewhale 或 cargo install codewhale-cli codewhale-tui --locked

扩展下载 · 详细卡片

Agent Kanban工头 #1

Agent-first 看板：Ed25519 身份、多 runtime（Claude Code/Codex/Gemini/ACP）、SSE 实时、In Review → 人 reject/complete。

冲突点：Web UI 只读，任务由 Agent CLI 创建 — 与 OpenX「用户立目标」相反。抄：状态机、SSE、review API、pnpm monorepo 结构。

  • vendors/agent-kanban/CLAUDE.md — 产品约束必读
  • vendors/agent-kanban/packages/
Mission Control工头 #2

OpenClaw 生态的编排台：SQLite、Kanban、多 Agent 适配、WebSocket/SSE、本机 pnpm start。

抄：自托管、零 Redis、任务看板。慎抄：32 面板、OpenClaw 绑定 — MVP 过重。

  • vendors/mission-control/openapi.json
  • vendors/mission-control/package.json
Gemini CLIExecutor · ACP

Google 官方终端 Agent；原生支持 ACP（--acp），与 JetBrains/Zed 生态互通。

OpenX：Executor 桥接协议首选调研对象之一（Client 在你们，Agent 在子进程）。Windows 克隆需 core.longpaths=true。

  • vendors/gemini-cli/packages/cli/
Aider执行层 · Python

最成熟的「结对编程」CLI 之一；git 集成、repo map、多模型。

OpenX：仅作 Executor 黑盒（spawn aider --message 类），勿用其架构做工头 UI。

  • vendors/aider/aider/
GooseBlock · 大而全

Block 开源 Agent：桌面应用、recipe、扩展；仓库体积最大（~347MB 浅克隆）。

OpenX：读扩展/recipe 思路即可；不建议作为第一 Executor。

OpenHands执行平台

软件工程 Agent：SDK、CLI、Local GUI、Docker 沙箱、子代理委托。

OpenX：明确不基于 OpenHands 建站；与「工头不干活」冲突。边界对照用。

  • vendors/openhands/README.md

OpenX 建议怎么用这些仓库

分层取用（不要混层）

OpenX 层优先参考做法
工头 UI + Goal API 自研 + agent-kanban、mission-control 抄 SSE/状态/review；UI 用 stage-design 原型
Executor 桥 Pi RPC/SDK、Gemini ACP、OpenCode server 抽象 POST /execute → 适配器子进程
多模型 openclaude、pi-ai 藏在 Executor 内，看板不展示品牌
不采用 openhands、aider 内核、langgraph 全栈 避免做成第四个 coding agent

Executor 适配优先级（本机 MVP）

  1. Pi --mode rpc — TS 同源、文档齐全、易嵌入 Node 服务
  2. ACP Client — 接 Gemini CLI / Codex 等（长期一套协议）
  3. OpenCode HTTP — 若需现成 server/session
  4. CodeWhale / OpenClaude — 备选 CLI spawn

团队阅读顺序（约 2 天）

  1. vendors/agent-kanban/CLAUDE.md + OpenX stage-design 原型
  2. vendors/pi/packages/coding-agent/docs/rpc.md + examples/sdk/
  3. vendors/opencode/packages/opencode/ 目录扫一眼 session 模型
  4. 选 1 个 CLI 在本地跑通，验证「工头 dispatch → 子进程 → progress 回调」
