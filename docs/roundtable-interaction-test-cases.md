# 圆桌功能交互测试用例

> 普通人使用调度台 / 项目对话圆桌的路径验证。  
> **断言重点**：席位、两行工具栏、灰显、@、发送管线、diverge 记忆。  
> **LLM 正文**：尽力验证；超时/无密钥记「环境受限」，不挡 UI 判定。

## 前置条件

- Web：`http://127.0.0.1:5173/`
- API：`http://127.0.0.1:3921/`
- 浏览器打开调度台，聊天面板（Chat）可见
- 不执行删项目、清库等破坏性操作

## 判定标准

| 层级 | 说明 |
|------|------|
| UI | 可见文案、class、disabled、计数、布局分行 |
| 管线 | 进入圆桌发送态（回复中 / 停止回答 / 席位气泡），非「仅圆桌可使用」类错 |
| LLM（可选） | 有可读回复正文 |

## 用例清单

### RT-01 工头态可见席位

1. 打开调度台，确保 Chat 面板可见  
**期望**：有工头占位 chip、`+ 添加`、计数约 `1/7`；无大块「圆桌已启用」提示条  

### RT-02 两行工具栏

1. 观察 Composer 工具栏  
**期望**：**上行**席位；**下行** Skill / MCP / 权限 / 知识；非席位与 Skill 错乱叠在同一视觉块  

### RT-03 加人开圆桌

1. 点 `+ 添加` → 选画像与模型 → 保存  
**期望**：席位 ≥2；计数更新；岛提示含「已开启圆桌」；可用 `@`  

### RT-04 Context 可点且进发送

1. 圆桌启用后点 Skill / MCP / 权限 / 知识并选中若干项  
2. 发送短句，检查 Network 中 `POST .../chat/rounds`  
**期望**：tab **可点**（未灰显）；请求 body 含对应 `skillIds` / `mcpIds` / `knowledge` / `permissionMode`  

### RT-05 无 @ 发送

1. 输入短句（如「你好，请简单介绍一下」）→ 发送  
**期望**：进入圆桌管线（回复中… / 停止回答 / 气泡）；非斜杠错误  

### RT-06 @ 点名

1. 输入 `@`  
**期望**：出现 mention 弹出层；可选成员；Enter/点击可选中  

### RT-07 diverge 记忆

1. 模式改为「发散（并行盲答）」→ 发送短句 → 再看模式  
**期望**：仍为「发散」，不被打回「定向」  

### RT-08 席位编辑关闭

1. 点非禁用席位 chip 打开编辑 → Esc 或点外侧  
**期望**：编辑器关闭  

### RT-09 工头态 Context（加人前）

1. 在尚无额外席位（或新建工头对话）时点 Skill  
**期望**：Skill/MCP **可点**（未灰显）  

> 注：若当前会话已是圆桌，本用例在项目对话新建工头会话上验证，或记录「当前会话已圆桌，跳过并在 RT-10 新建会话验证」。

### RT-10 项目对话同路径

1. 侧栏打开任意项目对话  
2. 核对席位条 + 两行工具栏 +（若未圆桌）可添加  
**期望**：与调度台核心表现一致（RT-01～RT-04 同类）  

### 可选

- **RT-11** 停止回答：发送后点「停止回答」  
- **RT-12** 静音后 `@` 候选不含该席
